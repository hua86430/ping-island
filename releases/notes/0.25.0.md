## 新增
- 通知橫幅：通知中心模式下，Claude 回覆完成時島會自動展開通知預覽，約 5 秒後自動收合；游標移入暫停倒數、移開即收合。
- 「懸停展開延遲」設定（0–1 秒，預設 0.24）：調整滑鼠懸停到島展開的等待時間。
- 「展開動畫時長」設定（0.15–0.8 秒，預設 0.42）：調整島展開／收合動畫速度，開合永遠同速。

## 修改
- 通知中心模式的未讀判定改以「助理回覆完成」為準（hook 家族的 Stop 事件、Codex 回合結束）：開新 session、自己輸入提示、工具執行過程都不再產生通知。
- 通知中心模式下，滑鼠懸停展開也顯示通知列表（原本仍顯示 session 清單）。
- 收合的島在通知中心模式只顯示未讀徽章，不再同時顯示 session 數量。
- 通知列改為兩行：第一行 session 標題與時間，第二行最新 LLM 回覆內容。

## 修復
- 開新 Claude session 時島誤彈出空白通知面板：純 prompt 就緒事件不再自動展開；提問／審批等需要行動的事件照常彈出並保持開啟。
- 通知面板展開後不會自動收合：補上 5 秒自動收合計時器，並修正兩個計時器掛載時序問題（`@Published` willSet 期間讀到舊狀態、注意力事件解除後面板失去計時器）。
- 通知面板高度不隨列數增長導致第二列被裁切。
- Codex 輔助執行緒可能漏入通知列表：閒置豁免僅適用「只因閒置而隱藏」的 session。
