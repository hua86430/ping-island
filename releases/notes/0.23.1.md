## Fixes
- Fixed historical Codex app-server threads being treated as newly completed work, preventing stale sessions from showing completion popups.
- Fixed stale tracked sessions re-queueing completion notifications when old completed, ended, or compacted states refresh without fresh activity.
- Filled a usage analytics gap for Claude-family sessions such as Claude and Qoder by reading token totals from transcript JSONL files.
- Let non-intervention session rows jump back to their owning AI agent terminal or IDE surface on double-click while preserving in-app response routing for approvals and questions.

## Notes
- The Sparkle build number is now `59`, and the app version is now `0.23.1`.
- This release is based on latest `main` after `v0.23.0` and focuses on Codex history notification correctness, Claude-family usage totals, and session jump-back behavior.
