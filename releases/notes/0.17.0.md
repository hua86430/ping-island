## Highlights
- Improved concurrent Claude `AskUserQuestion` handling: multiple questions in the same session are queued by `toolUseId` and answered in order instead of overwriting each other or closing sockets too early.
- Long Codex and Claude subagent content is now truncated only at display boundaries, keeping complete underlying data while preventing the Option+J detail panel from stalling on huge text.
- Short session preview rows now sit visually centered inside their stable reserved height, making newly-started or one-line sessions feel less top-heavy.

## Fixes
- Fixed stale Claude transcript-fallback question cards that could remain visible after the matching tool result completed.
- Fixed concurrent `AskUserQuestion` flows where only the newest question stayed visible and older pending responses could be orphaned.
- Fixed high CPU and energy use when very long subagent prompts or results were handed directly to SwiftUI Text / Markdown layout.

## Notes
- The Sparkle build number is now `49`, and the app version is now `0.17.0`.
- This release focuses on Claude question interventions, subagent detail rendering stability, and session-list preview polish.
