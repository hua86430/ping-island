# v0.25.6

統計頁改成逐模型計算用量與花費,並修正先前把快取重讀當成消耗、灌爆數字的問題。

## 新增

- 統計頁新增「各模型用量與花費」卡:上方一張每日花費多線趨勢圖(每個模型一條線,花費前 5 名 + 其餘併為「其他」),下方逐模型清單列出各模型的 token 與花費,依花費由高到低排序。
- 逐模型官方費率取代單一均價。每個模型套用各自的官方牌價計算花費:Opus 4.x 為每百萬 token 輸入 $5 / 輸出 $25、Sonnet 5 為 $2 / $10、Haiku 4.5 為 $1 / $5、Fable 5 為 $10 / $50、GPT-5.5 為 $5 / $30 等;無法辨識的模型才退回舊的均價估算。快取寫入以輸入價的 1.25 倍、快取命中以 0.1 倍計價。
- Codex 補上快取命中(`cached_input_tokens`)拆分,快取命中的部分以 0.1 倍計價,Codex 的花費估算更接近實際帳單。

## 修改

- 「Token 費用預估」的花費改用逐模型官方費率加總計算,比舊的單一均價更準(舊均價會低估 Opus、高估 Haiku)。當有逐模型資料時,卡片下方標示改為「按模型官方定價」。升級前既有的舊資料沿用均價估算、不會歸零。

## 修復

- 統計頁的「Token 消耗」與費用不再把快取重讀(cache-read)當成新消耗累加。先前每一輪對話都會把整段被快取的上下文重複計入,導致 token 數灌到十億級、費用估算暴衝到數百美元;現在消耗只計輸入、快取寫入與輸出,快取重讀單獨以 0.1 倍計價,數字回到合理範圍。
