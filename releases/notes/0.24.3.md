## 修復
- 修復同一目錄下、位於不同終端機的兩個 Claude Code 工作階段在通知列被合併成一行的問題。現在會依終端機身分（tmux pane / iTerm / 終端機工作階段識別碼，退回時用 tty）區分，不同終端機的工作階段會各自保留獨立一行、各自的完成提醒與點擊聚焦目標；同一終端機內重新啟動 Claude 仍會合併成一行。

## 說明
- 通知列去重現在在 UI 層（`SessionMonitor`）與儲存層（`SessionStore` 的孤兒工作階段清理）使用同一套終端機身分，兩層行為維持一致。
- 僅影響 Claude 工作階段；Codex 等其他 provider 的去重邏輯不變。
- Sparkle build 號提升到 `63`，版本號提升到 `0.24.3`。

## Fixes
- Fixed two Claude Code sessions in the same directory but different terminals collapsing into a single notch row. Sessions are now distinguished by terminal identity (tmux pane / iTerm / terminal-session id, falling back to tty), so different-terminal sessions each keep their own row, completion notification, and click-to-focus target; relaunching Claude in the same terminal still collapses to one row.

## Notes
- Notch dedup now uses the same terminal identity at both the UI layer (`SessionMonitor`) and the store layer (`SessionStore` orphan cleanup), so the two stay consistent.
- Only Claude sessions are affected; dedup for Codex and other providers is unchanged.
- The Sparkle build number is now `63`, and the app version is now `0.24.3`.
